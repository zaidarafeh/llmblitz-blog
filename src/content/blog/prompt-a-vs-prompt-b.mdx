You've been there. Two prompts. One output looks better than the other. You ship the better one and move on — never knowing *why* it was better, or whether it'll hold up when your inputs change.

We ran an experiment. Same task, two prompt variants. Not just comparing the words in the outputs — comparing the token-level confidence at every position. Here's what we found.

## The setup

**Task:** Classify customer support tickets into one of five categories.

**Prompt A:**
```
You are a helpful assistant. Please read the following support ticket and classify it into one of these categories: billing, technical, account, shipping, or general. Be accurate and concise.
```

**Prompt B:**
```
Classify the support ticket below. Return only one word from: billing, technical, account, shipping, general.
```

Both prompts return the right category most of the time. Prompt B is shorter. That's all most people would notice.

## What the token data revealed

{/* SCREENSHOT: BlitzLab compare mode output — token confidence heatmap for Prompt A vs B */}

When we ran both through BlitzLab's comparison mode, something jumped out immediately.

Prompt A generated an average token confidence of **61%** on the output. Prompt B: **89%**.

The difference wasn't in the category word itself — both prompts produced it with high confidence. The difference was in everything *before* it. Prompt A produced low-confidence hedging tokens before the answer: words like "The" and "Based" and "I" — the start of an explanation the model was about to give before catching itself.

**Finding:** The phrase "Be accurate and concise" in Prompt A actually *increased* uncertainty. It signals to the model that there's a dimension of quality to evaluate, prompting it to begin a longer response before self-correcting.

## The specific token that gave it away

Token 3 in Prompt A's output — the word "category" (as in "The category is...") — had a confidence of **43%**. The same position in Prompt B had confidence **94%**.

That 51-point gap at a single token is the model deciding: *am I explaining this, or just answering?* Prompt A left it ambiguous. Prompt B removed the ambiguity entirely.

## What this means for your prompts

The lesson isn't "shorter is better." It's that **instructions that describe desirable qualities** ("be accurate", "be concise", "be helpful") tend to add uncertainty rather than reduce it. They're meta-instructions that make the model second-guess its output strategy.

The instructions that work are ones that **constrain the output format** directly. "Return only one word" is not a quality instruction — it's a structural constraint. The model knows exactly what it's producing before it produces the first token.

## The rule of thumb

If you have a prompt that "mostly works" but occasionally produces longer, messier outputs than you want — look for phrases like:
- "Be accurate and concise"
- "Make sure to..."
- "Please note that..."
- "Try to..."

These are often the source of the low-confidence tokens right before your answer. Replace them with output format constraints and re-run.
