When you're running 10,000 LLM calls a month, a prompt that's 30% cheaper isn't a nice-to-have — it's $300 you keep. Most teams never look this closely. Here's what you find when you do.

## Two prompts, same job

Both prompts were used for the same task: extracting a structured summary from a user-submitted review.

**Prompt A (the expensive one):**
```
You are a professional assistant helping an e-commerce team understand customer sentiment. 
Your task is to carefully read the customer review below and provide a detailed analysis.

Please make sure to:
- Identify the overall sentiment (positive, negative, or neutral)
- Extract the key topics mentioned by the customer
- Note any specific product features that were praised or criticized
- Summarize in 2-3 sentences

Be thorough and accurate. The team will use this analysis to improve the product.
```

**Prompt B (the optimized one):**
```
Extract from the review below:
1. Sentiment: positive / negative / neutral
2. Topics: [comma-separated list]
3. Features praised: [list or "none"]
4. Features criticized: [list or "none"]
5. Summary: [2 sentences max]

Return as a numbered list. No explanation.
```

Same task. The outputs are equivalent for the team's purposes.

At 100k calls/month, that's **$11.80/month saved per model** — and you're likely running this across multiple models or environments.

## Where the tokens went

The token difference breaks down into two buckets:

### 1. Input bloat (−195 tokens)

Prompt A uses 290 input tokens. Prompt B uses 95. Here's where the extra 195 went:

- **"You are a professional assistant helping an e-commerce team..."** — 18 tokens of context the model doesn't need to do the job
- **"Please make sure to:"** and **"Be thorough and accurate."** — 14 tokens of meta-instruction that add no behavioral constraint
- **Verbose bullet descriptions** — "Note any specific product features that were praised or criticized" vs "Features praised: [list]" is a 9-token difference per instruction × 4 instructions

Each of these deletions sounds risky. None of them changed the output quality in our testing.

### 2. Output bloat (−110 tokens)

Prompt A doesn't constrain the output format, so the model fills in explanation:

> *"Based on the review, the overall sentiment is positive. The customer mentioned several key topics including..."*

That framing before the actual content costs ~40 tokens per call and adds nothing. Prompt B's numbered format forces the model to start the answer immediately.

## The optimization that paid off most

The single highest-impact change was adding **"No explanation."** at the end of Prompt B.

Without it, even the structured format prompt occasionally generated a one-sentence preamble. With it, every call started directly with "1. Sentiment:". That one addition removed ~25 tokens of output consistently.

At 100k calls/month on GPT-4o mini: $0.60/1M output tokens × 25 tokens × 100,000 calls = **$1.50/month from two words**.

## The quality tradeoff (there isn't one here)

We checked. We rated 20 paired outputs for: completeness, accuracy, and usefulness to the team.

{/* SCREENSHOT: EcoBlitz output comparison — side-by-side outputs for Prompt A vs B */}

Prompt B matched or exceeded Prompt A on all three dimensions. The verbose framing in Prompt A wasn't adding quality — it was adding overhead.

This isn't always the case. For open-ended generation tasks, constraints can hurt quality. But for extraction and classification — the most common production LLM tasks — structured prompts almost always win on both cost *and* quality.

## How to find your own 4x prompt

The pattern we see repeatedly in production prompts:

1. **Context paragraphs** that explain *why* the task matters ("The team will use this to...") — the model doesn't need to know
2. **Quality meta-instructions** ("be thorough", "be accurate") — these add uncertainty, not quality
3. **Unconstrained output** — no format spec means the model writes an essay when a list would do
4. **Politeness phrases** — "please", "thank you", "I'd like you to" — 2-4 tokens each, zero behavioral impact
